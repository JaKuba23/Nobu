# Security Policy

## Responsible Use

Nobu is a network security tool intended for educational purposes and authorized testing only.

### Authorized Use

- Scanning your own networks and systems
- Penetration testing with explicit written permission
- Educational and research purposes in controlled environments
- Network troubleshooting on systems you own or administer

### Prohibited Use

- Scanning networks or systems without authorization
- Using the tool for malicious purposes
- Circumventing security controls without permission
- Any activity that violates applicable laws

## Legal Notice

Unauthorized port scanning may violate laws in your jurisdiction, including:

- **United States:** Computer Fraud and Abuse Act (CFAA)
- **United Kingdom:** Computer Misuse Act 1990
- **European Union:** Various national cybersecurity laws
- **Poland:** Kodeks Karny (Art. 267-269)

Always ensure you have proper authorization before scanning any network.

## Reporting Security Vulnerabilities

If you discover a security vulnerability in Nobu:

1. **Do not** open a public issue
2. Send a private report to the maintainer
3. Include detailed steps to reproduce
4. Allow reasonable time for a fix before disclosure

### What to Report

- Vulnerabilities in the scanning engine
- Issues that could allow unauthorized network access
- Bugs that could cause unintended system impact
- Problems with input validation that could be exploited

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Fix Development:** Depends on severity and complexity
- **Disclosure:** Coordinated with reporter

## Security Best Practices

When using Nobu:

1. **Get Permission First** - Always have written authorization
2. **Know Your Scope** - Only scan authorized targets
3. **Monitor Impact** - Watch for unintended effects
4. **Secure Your Results** - Protect scan data appropriately
5. **Follow Policies** - Adhere to organizational security policies
